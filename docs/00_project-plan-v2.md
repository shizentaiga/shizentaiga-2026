## 1. プロジェクトの目的と概要
本プロジェクトは、外部プラットフォーム（リザスト/PayPal）に依存している現状の shizentaiga.com を、**Cloudflare Pages + Hono (TypeScript)** によるモダンな動的フルスタック環境へ移行し、ビジネスロジックの完全内製化を実現することを目的とする。

### 主な変更点
- **決済・予約の内製化**: Stripe Checkout を採用し、自社ドメイン内でのセキュアな予約・決済フローを構築。外部プラットフォームの制約を排除する。
- **データガバナンスの強化**: Cloudflare D1 (SQLite) を活用し、顧客対応履歴や予約データの一元管理と高速なレスポンスを実現する。

## 2. 開発・移行戦略（リスク管理）
現行サイト（Project A）の稼働を維持したまま、新環境（Project B）を構築する**「ブルー・グリーン・デプロイメント」**形式を採用し、ダウンタイムおよび移行リスクを最小化する。

### 移行ステップと検証項目
1. **Hono 環境構築**: TypeScript による厳密な型定義を行い、APIの堅牢性を担保する。
2. **既存資産の完全移植**: `favicon`、`style.css`、`JSON-LD` 等、SEOおよびブランドアイデンティティに関わる資産を無加工で統合。
3. **Stripe サンドボックス検証**: テスト環境（`v2.shizentaiga.com` 等）にて、決済Webhookの到達確認および例外処理の挙動を検証。
4. **DNS 切り替えと死活監視**: Cloudflare DNS による切り替えを実施。不具合検知時には1分以内に旧環境（静的HTML構成）へ切り戻すプロトコルを策定済み。

## 4. プロジェクト間の独立性と技術シナジー
本プロジェクトは、現在開発中の **「Aletheia (アレテイア)」** とは、リポジトリおよびデータベースを完全に分離した独立プロジェクトとして定義する。

- **Aletheia プロジェクト**: 一般事業者向け予約・決済 SaaS プラットフォーム開発。
- **shizentaiga.com V2**: 公式サイトにおける、特定業務（コンサルティング・コーチング）に最適化した独自実装。

### シナジー（R&D 側面）
公式サイト V2 で確立した「Hono + D1 + Stripe」のアーキテクチャおよび決済エラーハンドリングの知見は、将来的な Aletheia プロジェクトの信頼性を担保するための**概念実証（PoC）**として機能する。

## 5. 監査およびセキュリティ上の留意事項

### 5.1 決済情報の非保持化
Stripe の **Elements / Checkout** を利用することで、カード情報は自社インフラを通過・保存させず、Stripe サーバーへ直接送信される。これにより、PCI DSS の簡略化された自己問診（SAQ-A）に準拠する構成とする。

### 5.2 法的遵守と透明性
本日（2026/04/05）定義した `/legal` ディレクトリにおける「特定商取引法に基づく表記」および「プライバシーポリシー」との整合性を、実装の各フェーズで再検証する。

### 5.3 データバックアップ
D1 データベースの自動バックアップ機能を有効化し、災害復旧（DR）体制を整える。
