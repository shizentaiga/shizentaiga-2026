■ 3. 実装上の重要注意点（経営コンサルタントの考察）

* 送信ドメインの固定化
  認証完了後は、必ず「@shizentaiga.com」ドメインを使用すること。resend.devドメインはテスト用であり、実運用では到達率やブランド毀損のリスクがあるため、コード内でのFrom指定は定数化を推奨。

* 環境変数の秘匿（セキュリティ）
  APIキー（re_...）は、 wrangler.toml や Cloudflare Dashboard の Secrets に保存し、ソースコードへの直書きを厳禁とする。

* 到達率の維持（レピュテーション管理）
  Resendは到達率が高いサービスだが、短時間に大量の「予約キャンセル→再予約」などの通知が飛ぶとスパム判定されるリスクがある。本運用の際は、送信ログを定期的に監視し、バウンス（不達）が発生していないかチェックする運用フローを検討すること。

スモールスタート期においては、ランニングコスト・ゼロで最高品質の通知体験を提供可能。
