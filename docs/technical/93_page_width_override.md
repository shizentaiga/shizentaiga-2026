# 技術資料：共通レイアウトの制約下における個別ページのスタイル拡張

---

## 1. 今回起こった問題

共通レンダラー（`renderer.tsx`）において、`.container` クラスで `max-width: 540px`（黄金比）の制限をかけていた。

特定商取引法に基づく表記など「表（テーブル）」を多用する法務ページ（`Legal.tsx`）では、540px では横幅が狭すぎて視認性が著しく低下するという競合が発生した。

| 競合の構造 | 内容 |
|:---|:---|
| 共通レイアウトの壁 | 親要素が物理的に 540px に固定されている |
| 個別ページの要求 | 子要素が 800px になりたいと主張しても、親の物理的な幅を超えられない |

---

## 2. 解決方法の比較

### ❌ 避けるべき手法：`!important` による強制上書き

CSS の詳細度を無視して強制的に上書きする手法。

一時的には解決するが、CSS の設計（詳細度の階層）を破壊するため、後のメンテナンスで予期せぬ挙動を招く。構造的解決が可能な場合は「邪道」とされ、技術的負債となる。

### ✅ 採用すべき手法：ID セレクタによる詳細度の制御

親要素に特定の ID を付与し、その ID を起点にスタイルを上書きすることで、`!important` を使わずに自然な CSS の優先順位で解決する。

---

## 3. 本質的でクリーンな解決策

### Step 1：`renderer.tsx`（親）に ID を付与

共通の器となる `div` に、外部からアクセスするための「目印（フック）」として ID を追加する。

~~~tsx
// renderer.tsx（修正例）
<div id="layout-root" className="container">
  {children}
</div>
~~~

### Step 2：`Legal.tsx`（子）で親の制限を書き換える

個別ページ側で、そのページが表示されている時だけ有効なスタイルを注入する。

~~~tsx
// Legal.tsx（修正例）
<style>
  /*
   * ID（#layout-root）とクラス（.container）を組み合わせることで、
   * 外部 CSS の .container 設定より詳細度が高くなり、自然に上書きされる
   */
  #layout-root.container {
    max-width: 800px;
  }
</style>

<div>
  {/* コンテンツ（ここでは 800px が適用される） */}
</div>
~~~

---

## 4. 今後の対応に向けたアドバイス

**「親の壁」を確認する**
子要素にスタイルを指定しても反映されない場合、まずその外側の要素（親・先祖）に `max-width` や `overflow` の制限がないかを確認する。

**ID を「鍵」として使う**
共通設定を一部のページだけで上書きしたい場合、共通部分に ID を振っておくことで、個別ページ側から「この ID の中にある時だけ特別扱い」という指定が安全に行える。

**`!important` は最後の手段**
スタイルが効かない時は、詳細度が足りないことがほとんど。「クラスを重ねる」か「ID を親に付ける」ことで、CSS のルールに則った解決を優先する。

---

## Claude からの補足アドバイス

### この手法の「射程」を正確に把握しておく

今回の `#layout-root.container { max-width: 800px; }` は**そのページが描画されている間、`layout-root` 全体に作用する**。`Legal.tsx` のスタイルタグはページ内の `<style>` として注入されるため、原則として `Legal.tsx` がマウントされている間だけ有効であり、他のページには影響しない。ただし HTMX でページを部分差し替えする構成では、`<style>` タグが DOM に残り続けるケースがある。その場合は `Legal.tsx` を離れた後も `800px` が維持されてしまう副作用が起きうるため、HTMX でのページ遷移が発生する箇所では動作確認を一度行っておくことを推奨する。

### `max-width` 以外の「親の壁」に注意

今回は `max-width` が原因だったが、同種の問題は以下のプロパティでも起きやすい。頭に置いておくと次回の原因特定が速くなる。

| プロパティ | 起きやすい症状 |
|:---|:---|
| `overflow: hidden` | 子要素が親からはみ出せず切れる |
| `position: relative` + `z-index` | 子の重なり順が意図通りにならない |
| `display: flex` の `flex-shrink` | 子要素が予期せず縮む |
| `padding` の内側計算（`box-sizing`） | 幅指定が実寸と合わない |

### 将来的な設計の選択肢

ページごとに `max-width` が異なるケースが増えてきた場合、`renderer.tsx` 側で `containerSize?: 'sm' | 'md' | 'lg'` のような props を受け取る設計に移行すると、個別ページ側からスタイルを注入する必要がなくなり、よりクリーンになる。

~~~tsx
// 将来的な renderer.tsx の設計例
const maxWidthMap = { sm: '540px', md: '720px', lg: '800px' };

<div
  id="layout-root"
  className="container"
  style={{ maxWidth: maxWidthMap[containerSize ?? 'sm'] }}
>
  {children}
</div>
~~~

現時点ではオーバーエンジニアリングになるため、今の ID フック方式で十分。ただし同様の上書きが3ページ以上に広がり始めたら、この方向へのリファクタリングを検討するタイミングと考えてよい。
