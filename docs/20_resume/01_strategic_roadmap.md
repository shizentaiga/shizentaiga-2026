# 個人事業（システム開発受託）活動方針・実行計画書
**〜Edge/Serverlessスタックを軸とした、高パフォーマンス技術支援戦略〜**

## 1. 活動コンセプト：技術をビジネス価値へ変換
単なる実装スキルの提供ではなく、「小規模〜中規模サービスの初期・改善フェーズにおける、コスト最小化とUX最大化の両立」を自らのポジショニングとする。

* **課題解決の言語化:**
    * **コストの最適化:** Cloudflare構成により、AWS等で懸念される予算超過リスクを排除し、インフラ固定費を極小化する。
    * **UXの最大化:** Hono/TypeScriptによる高速レスポンスと、PSI/Lighthouse 95以上を標準とする実装で離脱を防ぐ。
    * **堅牢なロジック:** 冪等性を考慮したDB設計、Stripe/Resend等を用いた信頼性の高い決済・通知基盤の構築。

## 2. 再現性を担保する「支援パッケージ」
クライアントが「自社のプロジェクトに適合するか」を即座に判断できるよう、支援内容を型化する。

### 【再現可能な支援内容】
以下の条件において、最短・最安・最高品質の結果を一貫して再現する。
* **対象:** 新規事業の立ち上げ、または既存サービスの特定機能切り出し。
* **技術:** Cloudflare Workers / D1 / Hono / TypeScript / Stripe。
* **プロセス:** 1. 要件整理（1-2日）
    2. 最小構成での設計提示
    3. 2週間以内の初期デプロイ（MVP開発）
    4. 計測・改善（PSI / UX）

### 【非対応領域（適用外条件）】
専門領域に特化し、確実に成果を出すため、以下の対応は原則として対象外とする。
* 数百万MAU規模など、大規模トラフィック前提の分散システム設計。
* 既存の巨大なモノリス・レガシーシステムの全面リプレイス。
* ※0→1および小〜中規模（初期〜改善フェーズ）の最適化に特化することで、高いパフォーマンスを保証する。

## 3. 稼働・契約の方針
既存ビジネスへの影響を考慮し、持続可能なワークスタイルを維持する。

* **稼働ボリューム:** 月間60〜70時間（週2日相当）。
* **契約形態:** **技術支援（準委任契約）を最優先。** 仕様変更への柔軟な対応と、ご自身の稼働時間のコントロールを両立させるため。
* **コミュニケーション:** 原則リモート、非同期（チャット）中心。電話は必要最低限に限定。
* **単価指標:** 月間60〜70時間：20万円〜40万円（週5稼働換算で月単価100万円目安）。

## 4. プラットフォーム別の戦い方

* **Wantedly（最優先・攻め）**
    「2週間で決済・予約基盤を構築」という再現性をフックに、スピードを重視するスタートアップの0→1フェーズへ直接アプローチする。
* **Findy / LAPRAS（受け・補強）**
    GitHubのREADMEに構成図と「支援プロセス」を明記。単なる技術スタックの羅列ではなく、仕事の進め方の「型」を見せる。
* **YOUTRUST（中長期・信頼）**
    「0→1専門の技術支援・顧問」としての認知を広め、紹介のハードルを下げる。

## 5. 具体的な活動計画（1ヶ月ロードマップ）

### 第1週：ポートフォリオの「課題解決型」へのリライト
* 既存実績（予約・決済システム）を事例に、「どのようなプロセスで、何日間で、どのスコアを実現したか」をテンプレート化して資料にまとめる。

### 第2週：プロフィールの一斉更新
* 各媒体に「再現可能な支援内容」と「非対応領域」を追加。「この条件なら確実にハマる人」と直感させる内容にリライトする。

### 第3週：ターゲット選定と面談開始
* Wantedly等で「新規事業」「0→1」タグの案件を自ら確認。週3〜5件程度のカジュアル面談を実施し、非同期文化の有無を確認する。

### 第4週：契約締結・稼働開始
* 条件に合致する案件（月60〜70h・技術支援）での契約締結。
