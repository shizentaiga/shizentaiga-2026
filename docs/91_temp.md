# Aletheia Core：実装フェーズ・ロードマップ

## 1. データ操作層 (Database Logic) の構築
D1 のテーブルに命を吹き込むフェーズです。
- **DB初期化**: `schema.sql` をローカル D1 に適用。
- **クエリ実装**: `src/db/queries.ts` にて、予約枠の取得（getSlots）とアトミックな仮確保（tryLockSlot）のロジックを実装。
- **検証**: 実際にダミーデータを入れ、同時リクエストで「二重予約」が防げるかターミナルからテスト。

## 2. Stripe Checkout と Webhook の接合
「お金」と「在庫」を連動させるフェーズです。
- **セッション発行**: `src/lib/stripe.ts` で、31分（1860秒）期限付きのセッション作成関数を実装。
- **Webhook 受信**: `/api/webhook` を作成し、署名検証と「べき等性テーブル」への記録を実装。
- **検証**: Stripe CLI を使い、ローカル環境で「決済完了通知」を飛ばして、DB の status が自動で booked に変わることを確認。

## 3. 自己修復 (Cron) と通知の実装
システムの「自律性」を完成させるフェーズです。
- **Cron ロジック**: 35分経過した pending 枠を Stripe API に照合し、開放または確定させる Cron 関数を実装。
- **通知連携**: 予約確定時に Google Calendar へ登録し、Resend 等で確認メールを飛ばす処理を追加。
- **検証**: 決済途中で画面を閉じ、5分〜35分後に枠が自動で available に戻る（または確定する）ことを確認。

## 4. UI への統合と本番デプロイ
ユーザーが触れる形に整える最終フェーズです。
- **予約カレンダー**: `/services` ページにて、D1 から取得した空き枠を動的に表示。
- **本番移行**: `wrangler d1 execute --remote` で本番 DB を初期化し、本番ドメインへデプロイ。
