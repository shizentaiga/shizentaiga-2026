# Git 切り戻し運用手順書 (Emergency Revert Guide) v2.0

**鉄則**: 原因調査（Debug）より先に、まず「動く状態」へ戻してサイトの平穏を取り戻すこと。

---

## 1. 逆引き切り戻しフロー

### ① ファイル単位で「一つ前」に戻す

**対象**: 特定のファイル（例: `renderer.tsx`）だけ1回分の変更をキャンセルしたい場合。

~~~sh
git checkout HEAD^ [ファイルパス]

# 現代的な Git (v2.23〜) での代替コマンド
git restore --source=HEAD^ [ファイルパス]
~~~

### ② ファイル単位で「二つ以上前」に戻す

**対象**: 数回修正を重ねたが、数日前の状態が最善だったと判断した場合。

~~~sh
git checkout HEAD~2 [ファイルパス]
# ~2 を ~3 にすれば3つ前、というように遡る数を指定できる
~~~

### ③ 特定のコミットにピンポイントで戻す

**対象**: 修正回数が多く、何個前か数えるのが難しい場合。

1. GitHub の「Commits」履歴から、戻したい時点の **7桁のハッシュ**（例: `a1b2c3d`）をコピー。
2. 以下を実行する。

~~~sh
git checkout [ハッシュ] [ファイルパス]
~~~

---

## 2. コマンドの意味とメカニズム

| 用語 | 意味 | イメージ |
|---|---|---|
| `checkout` | ファイルの取り出し | 過去の倉庫から特定の設計図を「今の手元」に持ってくる |
| `restore` | ファイルの復元 | Git v2.23〜 での「戻す」専用コマンド |
| `HEAD` | 最新の地点 | 「今、自分が立っている場所」を指す目印 |
| `^`（ハット） | 一つ前 | `HEAD^` で「今の場所のひとつ後ろ」を指す |
| `~n`（チルダ） | n個前 | `HEAD~2` で「今の場所から2ステップ後ろ」を指す |

---

## 3. 運用の鉄則

### ① 戻す前に現在の状態を避難させる

失敗したコードの中にも「正解のロジック」が含まれていることがある。
作業前に別ブランチへ退避させることで、後の原因調査が容易になる。

~~~sh
# 今の状態を別ブランチとして保存してから切り戻し作業に入る
git checkout -b fix/failed-attempt
~~~

### ② 必ず git commit を行い、事実を履歴に刻む

Cloudflare Workers 等の CI/CD 環境では、**新しいコミットが積まれないと再ビルドが走らない**。
ファイルを戻した後は必ずコミットし、「正常な状態に戻した」という事実をデプロイのトリガーにすること。

~~~sh
git add [ファイルパス]
git commit -m "revert: [ファイル名] を正常動作時点へ切り戻し"
git push origin main
~~~

### ③ ブラウザキャッシュを疑う

コードを戻しても直らない場合は、**シークレットモード**で動作確認を行うこと。

---

## 4. 応用：プロジェクト全体を戻す（緊急用）

直前の作業を完全になかったことにする。

~~~sh
git reset --hard HEAD^
~~~

> ⚠️ **警告**: 未コミットの作業も全て消えるため、**最終手段**として使用すること。
> 実行前に必ず `git checkout -b fix/failed-attempt` で退避を行うこと。

---

## 5. ナレッジ共有：「画像のみ表示」不具合のデバッグ指針

過去に発生した「トップページが画像のみになった」事象から得られた教訓。

### 推測される原因

| 原因 | 内容 |
|---|---|
| `renderer.tsx` の構造崩壊 | `children` が正しく渡されなかった |
| CSS の誤適用 | `display: none` 等がメインコンテンツに意図せず適用された |
| クラス削除の影響 | `.container` クラスの削除によりレイアウトの依存関係が崩れた |

### 次回の対策

`renderer.tsx` の HTML 構造を変更する際は、**一段ずつコメントアウトして影響範囲を確認する「消去法テスト」**をローカルで徹底すること。

~~~tsx
// 例：どこでレイアウトが崩れているかを特定する消去法
// <div class="container">
//   {children}
// </div>
~~~
